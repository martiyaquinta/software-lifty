# Plan: Passenger Data in IncomingRequestScreen & NavigationScreen (Issue #129)

## Global Constraints

- Backend: Bun + Elysia + Drizzle, tests with `bun test`
- Mobile: Expo SDK 54 + React 19 + TypeScript 6, theme at `src/theme/index.ts`
- Follow existing code patterns exactly
- All new fields nullable for backward compatibility
- Use `theme.colors.*`, `theme.radius.*`, `theme.spacing.*`, `theme.fontSize.*` consistently
- No unused imports, no dead code
- Pre-commit hooks (biome + commitlint) enforced via Lefthook
- Tests must pass: `bun run test` (backend) and `bun --filter @lifty/mobile test` (mobile)

## Tasks

### Task 1: Backend — Enrich trip API responses with passenger data

**Scope:** `apps/backend/src/features/trips/`

1. Update `tripService.getActiveTrip` to LEFT JOIN `users` on `passenger_id`
2. Add subquery for `passenger_rating` (AVG of `ratings.score` where `ratee_id = passenger_id`)
3. Return new fields: `passenger_name`, `passenger_avatar_url`, `passenger_rating`, `passenger_phone`
4. Update `tripService.getTripById` with the same enrichment
5. Validate: existing tests still pass, manual verification that new fields appear in response
6. Update mobile `tripSchema` in `apps/mobile/src/api/types.ts` with the 4 new fields (all `z.string().nullable()` / `z.number().nullable()`)

### Task 2: Mobile — Create Avatar and RatingStars components

**Scope:** `apps/mobile/src/components/`

1. Create `Avatar.tsx`: circular image with initials fallback
   - Props: `uri: string | null`, `name: string`, `size: number`
   - If uri: render `Image` with `borderRadius: size/2`
   - Else: render View circle with initials (first char of name, uppercase), `mediumGray` bg, `white` text
2. Create `RatingStars.tsx`: star display + numeric value
   - Props: `rating: number`, `size?: number` (default 14)
   - Stars: filled ★ for each integer, half-star or just numeric
   - Color: `amber`
   - Format: `★★★★★ 4.8`
3. Write tests

### Task 3: Mobile — Update IncomingRequestScreen with passenger data

**Scope:** `apps/mobile/src/screens/IncomingRequestScreen.tsx`

1. Import Avatar + RatingStars
2. Add passenger row inside dark modal card (above route info):
   - Avatar (size 48) + passenger_name + RatingStars
3. Add ETA to pickup: on mount, call `GET /maps/directions` with current location → pickup coords
   - Display: `~{etaMinutes} min al pickup`
4. Graceful fallback: if passenger data is null, show placeholder or hide
5. Update tests in `incoming-request.test.tsx`

### Task 4: Mobile — Add collapsible passenger card to NavigationScreen

**Scope:** `apps/mobile/src/screens/NavigationScreen.tsx`

1. Add collapsible passenger card above existing route card
2. State: `isExpanded: boolean`, toggled by tap
3. Collapsed: thin bar with small Avatar (32) + name
4. Expanded: Avatar (56) + name + RatingStars + passenger phone (optional)
5. Animate with `LayoutAnimation.configureNext(LayoutAnimation.Presets.easeInEaseOut)`
6. Styling: dark semi-transparent bg, rounded corners, consistent with IncomingRequestScreen
7. Graceful fallback for null passenger data
